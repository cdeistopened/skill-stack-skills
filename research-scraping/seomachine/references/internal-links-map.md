## Linking Best Practices

1. **Every article should link to:**
   - At least 2-3 internal pages
   - Info session or main site as CTA
   - 1-2 relevant external authority sources

2. **Anchor text guidelines:**
   - Use natural, descriptive anchor text
   - Vary anchor text (don't always use exact keyword)
   - Make it clear where the link goes

3. **Placement:**
   - First internal link within first 300 words
   - CTA link at end
   - Contextual links throughout body

4. **Hub & Spoke:**
   - Hub pages should link to all related spoke content
   - Spoke content should link back to hub
   - Cross-link between related spokes

## State Pages (Hub & Spoke)

**Architecture** (per Feb 2026 MTH SEO meeting):
- State landing pages (opened.co/indiana, etc.) = conversion-focused, hard sell
- "How to Homeschool in X" blog posts = higher funnel, capture search traffic
- Blog posts link TO state landing pages as primary CTA (virtual hub-and-spoke)
- State pages can bury a link back to the blog post, but not prominently
- Eventually: map hub page linking all state blog posts (hidden from main nav)
- Blog posts should NOT cannibalize state landing pages - different intent

## Guest Contributor Linking Strategy

Match guest expertise to keyword targets. The guest's audience drives initial traffic; SEO captures long-tail.

**Process:** Predetermine topic/keyword → check if guest fits → write in their voice using their published work → optimize for keyword. If keyword doesn't fit naturally, treat as newsworthy content (like dailies) rather than forcing SEO.

**Internal linking from guest posts:** Link to relevant hub page, tool page, or state page. Guest post → hub page → conversion page (3-click path).
