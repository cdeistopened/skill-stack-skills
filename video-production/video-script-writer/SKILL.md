# Video Script Writer

End-to-end video scriptwriting. Takes an idea from concept to camera-ready script in 5 steps. Based on Kallaway's scriptwriting framework, EVR (Expectations vs. Reality) psychology, value loop structure, and But/Therefore conflict method.

## When to Use

- Writing a YouTube video (long-form or short-form)
- Structuring a course lesson or webinar segment
- Turning raw bullet points into a watchable narrative
- Any spoken content that needs to hook, hold, and close

**Not for:** Writing hooks only (use `hook-writer`), podcast show notes (different structure), blog posts (different format), social captions.

---

## The EVR Principle (Core Psychology)

**Viewer happiness = Reality minus Expectations.**

If expectations are set by the title/hook and reality (the actual content) exceeds them, the viewer is satisfied. If reality falls short, they leave. Every step in this framework is designed to set expectations precisely and then beat them.

---

## Step 1: Packaging

The title and idea are the container. They set the expectation the script must beat.

**Process:**
1. Write the video idea as one sentence: what pain point does this solve, or what rabbit hole does this open?
2. Write a title that triggers a click curiosity loop. The title must make a promise the intro can confirm or exceed.
3. Suggest a thumbnail direction (what visual would pair with this title?).

**Rules:**
- The title is a contract with the viewer. Never promise what the script can't deliver.
- Title + thumbnail should create a single curiosity loop, not two competing ones.

---

## Step 2: Outline

Raw points become ordered, audited bullets.

**Process:**
1. Take the user's 3-8 raw bullet points and rewrite each as a clean one-liner
2. **Uniqueness audit:** For each point, ask: Has the audience heard this before? Is the tip itself unique, or is the distillation/example unique? Flag any point that's just common knowledge repackaged.
3. **Point ordering — the ascending value pattern:**
   - **Position 1:** Second-best point. Opens strong, sets a high bar.
   - **Position 2:** Best point. Exceeds the bar. The viewer thinks "if that was #1, what's #2?" and gets rewarded.
   - **Positions 3+:** Descending gradually. The viewer has already gotten the best value and stays for completeness.

**Why this order works:** The brain reads an ascending-then-gradually-descending pattern as "worth staying for." If the best point is last, viewers leave before reaching it. If it's first, everything after feels like a letdown.

---

## Step 3: Intro (The Hook)

The intro confirms the title's promise, establishes trust, and previews the structure.

**Use `hook-writer` for the opening line(s)** — any mode works (archetype, snapback, power-word). Then build the full 4-part intro:

**Part 1 — Immediate Context:** Bluntly state what the video is about. Confirm the click. Beat expectations if possible (title promise + something extra).

**Part 2 — Common Belief:** Restate what the audience currently believes about this topic. Build common ground so they feel you're speaking to their situation.

**Part 3 — Contrarian Take:** Offer a perspective that contradicts the common belief. This is the scroll-stop moment. The viewer stays because their assumption was challenged.

**Part 4 — Proof + Plan:** Why should they trust your contrarian take? Credentials, case studies, results. Then give a structured plan preview: "These are my 5 steps for..." / "Here's the framework I use..."

**Psychology tactics to apply:**
- **One Standard Deviation:** Pain point should be one step worse than what the viewer admits to. Not "your hooks could be better" but "your hooks are actively losing you subscribers."
- **Twist the Knife:** After naming the pain, make it sting. "And the worst part? You don't even notice it's happening."
- **Audience of One:** Write as if speaking to one specific person, not "creators" in general.

---

## Step 4: Body (Value Loops)

Each point in the outline gets three components. This is the meat of the script.

**The Value Loop (per point):**

| Component | What it does | How to write it |
|-----------|-------------|-----------------|
| **Context** | What this point IS | Explain as simply as possible. 6th-grade vocabulary. Define any term the viewer might not know. |
| **Application** | How to DO it | Concrete examples, case studies, demonstrations. "Here's exactly what that looks like..." |
| **Framing** | Why it MATTERS | Connect this point to the bigger narrative. Re-orient the viewer on the overall arc. |

**Between each point — Re-Hooking Transitions:**

Don't just move to the next point. Create urgency to hear it. Pattern:
- "That was the foundation. But if you skip this next step, everything we just built falls apart."
- "Most people stop here. The ones who break through do this next thing."
- "That's the easy part. Here's where it gets interesting."

The re-hook is a mini-curiosity-loop that keeps the viewer through the transition, which is the highest-drop-off moment in any video.

**Rhythm Rules:**
- Vary sentence lengths. Short. Medium ones that breathe a little. And then longer sentences that carry the reader through a complete thought before landing on a period. Check for the "jagged line" pattern — if all sentences are the same length, the script is monotone.
- Replace "and then" with "but" or "therefore." "And then" is additive (boring). "But" creates conflict. "Therefore" creates consequence. Both are interesting.
- Read aloud. If you stumble, rewrite.

---

## Step 5: Outro (High Note)

End strong. The outro has three jobs.

**1. Summary:** Recap what was covered. Remind the viewer their pain point is solved. Close the loop opened by the intro.

**2. The Last Dab:** A final line so memorable that if it's the only thing someone heard, they'd share it. Named after the Hot Ones finale — the moment everyone remembers. Write the last line first if needed (the "Last Dab Method" — start from where you want to end and work backwards).

**3. CTA (if applicable):** Embed it natively using one of two patterns:
- **Pain-then-solve:** Open a pain point the video didn't fully resolve, then offer the resource/product as the solve.
- **Speed-run-then-offer:** "We just covered the framework in 10 minutes. If you want to compress implementation to a weekend, here's [resource]."

Never break the content flow for a CTA. If it feels like an interruption, it is one.

## Quality Gates

| Check | Rule |
|-------|------|
| **Intro confirms title** | First 2 sentences must match the title's promise |
| **Point ordering** | Second-best in position 1, best in position 2 |
| **Value loop completeness** | Every body point has all 3 components (context, application, framing) |
| **Re-hooks present** | Every transition between body points has a re-hook |
| **Jagged line test** | Sentence lengths visibly vary throughout |
| **No "and then"** | Body transitions use "but" or "therefore" only |
| **Reading level** | 6th grade max for spoken content |
| **Last Dab exists** | Final line is standalone-memorable |
| **CTA is native** | If present, CTA doesn't break content flow |
